我为什么这么着迷大逃杀游戏

发表于 2018-07-11

最近痴迷大逃杀类型的有，比如开始的终结者、荒野行动，再到后来的刺激战场和全军出击。今天走在路上想了想，有以下这些原因：
1，逃避现实，对现实失望。曾经中学时我的视力急剧下降，打击太大，精神上崩了。到现在了，精神上还没有好转，羡慕不戴眼镜的人，羡慕有好视力的人，以至于疑神疑鬼草木皆兵，任何环境的变动都害怕对视力再次伤害。
2，朋友们都在玩，通过微信一拉就拉上来了。同学们很多玩刺激战场，还有发小们也在玩，当你一打开游戏本来就是想观看一会儿别人的游戏（里边可以观看别人玩），而朋友们已拉就把你拉到游戏里了。一玩就至少一小时（一局大概半小时）。
3，我骨子里有追求旷野之息的潜意识（也许，很多男的都有这种潜意识），而大逃杀的游戏给了灾难逃亡、荒野、孤独、荒凉、猎杀、对抗、狙击、拾荒、采集等体验。这些都满足了潜意识里的追求。我觉得人是被两部分控制的，一部分是大脑，一部分是你的基因（你的骨子里的东西），它们而在还在斗争。

20180831

C++中include时，有的带.h，有的不带，我上网查了一下原因。
统一C++各种后缀名，如.h、.hpp、.hxx等。标准化之前的头文件就是带后缀名的文
件，标准化后的头文件就是不带后缀名的文件。C++ 98 规定用户应使用新版头文件，对旧
版本头文件不在进行强制规范，但大多数编译器厂商依然提供旧版本头文件，以求向下兼
容。

另外，为了和C语言兼容，C++标准化过程中，原有C语言头文件标准化后，头文件名前带个
c字母，如cstdio、cstring、ctime、ctype等等

我们如果要用C++标准化了的C语言头文件，就得作如下的转换
#include <stdio.h> --> #include <cstdio>

C中
<.h>表示优先中系统目录中查找 该头文件 比如#include <stdio.h> 这是系统中的
“.h”表示优先从当前目录中查找 该头文件 比如#include “head.h” 这是你自己写的

20180830 const char ptr、char const ptr、char * const ptr

const char ptr是指针指向的内容不可以更改；
char const ptr同上；
char* const ptr是指针本身不可更改指向别的内容

20180813 C++的auto和decltype的学习

  • auto是C++11中声明变量时自动推测类型的
  • decltype也是C++11中声明变量时自动推测类型的
  • decltype和auto的语法不同，decltype可以使用表达式，该表达式在编译器不提前求值而是推测类型使用
  • 其次decltype和auto的区别还在于引用的声明时

我在内心里说自己是，工程师固执狂、怀旧、保守。一时之间对自己产生了消极否定的姿态。对defy手机的情节，木有原因，就是有。就像《大话》里边那句台词“需要吗？不需要吗？”。曾经在这个机器上，我做了n多次的刷机：从blur的android2.2刷到blur的android2.3，再刷到blur的android2.3.5，后来到搜狗，我对defy又刷了cm的4.0，再刷miui v5，再刷blur的android 2.3.5（中途经历过假板砖）。

Nexus 5固然配置高，屏幕也大，体验也好。defy配置低，512M的内存，800MHz的CPU。可是还是舍弃不了它，对它产生了一份情节。它代表了我在网秦工作的历史遗留物见证，看到defy，会想起来在网秦开发android app的场景，细节依然历历在目。而且会想起来一些同事。它也代表了2010年~2012年的android手机的高端机，那个时期的自豪。

我现在有两个手机Nexus 5，和defy(me525)。两个手机都是很喜爱。我幻想着如果有了自己的一个小房间，我会买n多个手机去把玩。买iphone我操，4s,买各种厂商的手机（华为、一加、小米、三星、锤子、vivo、find等）。或许我是一个手机搜集珍藏强迫症。
